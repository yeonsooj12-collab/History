# 정적 배포 안내

## 기본 원칙

세계사 조건 렌즈의 기본 모드는 수동 ChatGPT 복사·붙여넣기 방식입니다. 이 모드에서는 서버, 데이터베이스, 로그인, 빌드 도구, API 키가 필요 없습니다.

## 배포 루트

프로젝트 루트를 정적 사이트 루트로 사용합니다.

필수 파일과 폴더:

- `index.html`
- `styles.css`
- `js/`

선택 문서:

- `README.md`
- `docs/`

배포하지 말아야 할 파일:

- `.env`
- 개인 API 키 또는 토큰이 들어 있는 파일
- 로컬 테스트 로그

## 빌드 명령

필요 없습니다. 출력 디렉터리도 별도로 없습니다. 현재 파일 구조 그대로 정적 호스팅에 올립니다.

## GitHub Pages

일반 절차:

1. 저장소에 정적 파일을 커밋합니다.
2. `.env`가 커밋되지 않았는지 확인합니다.
3. GitHub 저장소의 Pages 설정에서 배포 브랜치와 루트를 선택합니다.
4. 배포 후 `index.html`, `styles.css`, `js/*.js`가 같은 상대경로로 로드되는지 확인합니다.

빌드 명령은 비워 둡니다.

## Netlify Drop

1. 프로젝트 폴더에서 `.env`를 제외합니다.
2. `index.html`, `styles.css`, `js/`를 포함한 폴더를 Netlify Drop에 올립니다.
3. 배포 URL에서 첫 화면과 수동 ChatGPT 흐름을 확인합니다.

빌드 명령은 필요 없습니다.

## Cloudflare Pages 등 정적 호스팅

권장 설정:

- Framework preset: 없음 또는 Static HTML
- Build command: 비움
- Output directory: 프로젝트 루트 또는 `/`
- Environment variables: 수동 모드만 배포할 경우 없음

## 수동 모드에서 서버가 필요 없는 이유

앱은 브라우저에서 입력을 정리하고 ChatGPT 요청문을 생성합니다. 사용자가 직접 ChatGPT에 붙여 넣고, 반환된 JSON을 다시 앱에 붙여 넣습니다. 이 과정에서 웹페이지가 자동으로 외부 API에 입력을 전송하지 않습니다.

## apiAi 사용 시 server.js가 필요한 이유

브라우저 JavaScript에 OpenAI API 키를 넣으면 키가 노출됩니다. `?apiAi=1` 모드는 로컬 또는 별도 서버의 `/api/interpret`를 통해 요청해야 합니다. 이때 `server.js`가 `.env`의 `OPENAI_API_KEY`를 읽고 OpenAI API를 호출합니다.

정적 호스팅만으로는 `/api/interpret`가 제공되지 않습니다. API 자동화를 배포하려면 별도 서버 또는 서버리스 함수로 옮겨야 합니다.

## 배포 후 smoke test

1. 첫 화면에 `세계사 조건 렌즈`가 보이는지 확인합니다.
2. 선택 상자와 직접 입력이 화면 밖으로 나가지 않는지 확인합니다.
3. `ChatGPT에 세계사 렌즈 요청 만들기`를 눌러 로컬 편집 브리프가 표시되는지 확인합니다.
4. 전체 요청 복사 버튼을 눌러 성공 문구 또는 직접 복사용 textarea가 제공되는지 확인합니다.
5. 정상 `axis-finder` JSON을 붙여 넣어 역사적 쟁점 카드가 표시되는지 확인합니다.
6. 쟁점 카드에서 `이 쟁점을 세계사로 읽기`를 눌러 2차 요청을 복사합니다.
7. 정상 `editor` JSON을 붙여 넣어 최종 결과가 표시되는지 확인합니다.
8. 잘못된 JSON을 붙여 넣었을 때 원문이 지워지지 않고 수정 가능한 오류 안내가 나오는지 확인합니다.
9. 모바일 너비에서 가로 스크롤이 불필요하게 생기지 않는지 확인합니다.
10. 브라우저 콘솔에 치명적 오류가 없는지 확인합니다.
