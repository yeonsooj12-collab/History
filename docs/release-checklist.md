# 릴리스 체크리스트

## 브랜드

- [ ] 제품명이 `세계사 조건 렌즈`로 보인다.
- [ ] 이전 명칭이나 임시 프로젝트명이 사용자 화면에 남아 있지 않다.
- [ ] 제품 설명이 조건 기반 세계사 비교 학습 도구라는 목적을 설명한다.

## 입력

- [ ] preset 선택이 유지된다.
- [ ] 직접 입력 원문이 보존된다.
- [ ] 조건 하나만 적거나 골라도 요청할 수 있다는 안내가 첫 입력 화면에서 보인다.
- [ ] 0단계 탐색 범위와 2~3단계 보조 입력은 비워도 된다는 점이 진행 표시와 문구에서 드러난다.
- [ ] 입력 변경 시 이전 결과가 stale 상태로 표시된다.
- [ ] 초기화 후 입력과 오류 상태가 정리된다.

## 1차 복사

- [ ] `ChatGPT에 세계사 렌즈 요청 만들기` 후 로컬 편집 브리프가 표시된다.
- [ ] 일반 ChatGPT용 전체 요청을 복사할 수 있다.
- [ ] 복사 실패 시 직접 선택 가능한 fallback textarea가 열린다.

## 1차 JSON 적용

- [ ] 정상 `axis-finder` JSON을 적용할 수 있다.
- [ ] ```json 코드펜스가 있어도 적용된다.
- [ ] 일반 ``` 코드펜스가 있어도 적용된다.
- [ ] `editor` JSON을 1차 칸에 넣으면 고칠 수 있는 오류가 표시된다.
- [ ] 오류가 나도 붙여 넣은 원문이 지워지지 않는다.

## 축 선택

- [ ] 역사적 쟁점 카드가 2~3개 표시된다.
- [ ] 각 카드에 제목, 핵심 인사이트, 반복되는 긴장, 사용 요소, 보류 요소, 가정, 후속 질문이 표시된다.
- [ ] 쟁점 선택 후 2차 학습 프롬프트가 선택한 쟁점에 맞게 생성된다.

## 2차 복사

- [ ] `이 쟁점을 ChatGPT에서 더 읽기` 버튼이 2차 학습 프롬프트를 복사한다.
- [ ] 2차 프롬프트는 JSON을 요구하지 않는다.
- [ ] 선택한 쟁점이 바뀌면 선택 상태와 복사 안내가 새 쟁점에 맞게 바뀐다.
- [ ] `쟁점 선택 취소`가 선택 상태를 초기화한다.
- [ ] 2차 답변 붙여넣기 textarea가 렌더링되지 않는다.

## ChatGPT에서 이어 읽기

- [ ] 2차 답변은 ChatGPT 창에서 바로 읽도록 안내된다.
- [ ] 앱에는 1차 `axis-finder` JSON만 붙여 넣는다고 안내된다.
- [ ] 참고문헌이나 각주가 필요한 긴 해설은 ChatGPT 대화창에서 이어질 수 있다.

## 오류 처리

- [ ] 잘못된 JSON 텍스트가 앱을 깨지 않는다.
- [ ] 완전 빈 입력일 때 하나 이상 입력하라는 안내가 나온다.
- [ ] 예상하지 못한 추가 필드가 있을 때 오류가 표시된다.
- [ ] `[object Object]`, `undefined`, `TypeError`가 사용자 안내로 노출되지 않는다.

## 모바일

- [ ] 360px 너비에서 가로 스크롤이 불필요하게 생기지 않는다.
- [ ] 390px 너비에서 버튼 글자가 잘리지 않는다.
- [ ] 768px 너비에서 방향 카드 순서가 자연스럽다.
- [ ] 1차 JSON textarea는 필요한 경우 내부 스크롤을 사용한다.

## 키보드와 접근성

- [ ] 주요 조작은 실제 `button`, `select`, `textarea`로 동작한다.
- [ ] 입력에는 label 또는 설명 텍스트가 연결되어 있다.
- [ ] 키보드만으로 주요 흐름을 사용할 수 있다.
- [ ] 오류와 성공 상태가 텍스트로 표시된다.
- [ ] 상태 안내 영역에 필요한 aria-live가 유지된다.

## 초기화와 stale

- [ ] 입력 변경 후 이전 AI 결과가 stale로 표시된다.
- [ ] 1차 결과 적용 후 초기화하면 결과와 오류가 제거된다.
- [ ] 오래된 쟁점 선택 상태가 다른 입력에 남지 않는다.

## 정적 배포

- [ ] `index.html` 기준 상대경로가 정상이다.
- [ ] 수동 모드는 `server.js` 없이 동작한다.
- [ ] `apiAi`와 `mockAi`는 기본 활성화되지 않는다.
- [ ] 외부 CDN이나 빌드 도구가 필수 아님이다.

## 비밀정보

- [ ] `.env`는 커밋 또는 정적 배포 대상에서 제외된다.
- [ ] `.env.example`에는 실제 키가 없다.
- [ ] 클라이언트 JavaScript에 API 키가 없다.
- [ ] 사용자 입력은 복사 전 외부 API로 자동 전송되지 않는다.

## 콘솔 오류

- [ ] 첫 로드 시 브라우저 콘솔에 치명적 오류가 없다.
- [ ] JSON 오류 입력 후에도 콘솔에 치명적 오류가 없다.

## 문서

- [ ] README가 공개 사용자용 순서로 정리되어 있다.
- [ ] `docs/deployment.md`가 정적 배포 절차를 설명한다.
- [ ] `docs/release-checklist.md`가 릴리스 전 점검 항목을 포함한다.
- [ ] 품질 조사 문서는 삭제하지 않았다.
