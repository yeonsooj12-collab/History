# 수동 ChatGPT 세계사 렌즈 워크플로

## 도구의 목적

세계사 조건 렌즈는 사용자가 고른 사회 압력, 외부 조건, 행위자, 영향을 받는 집단, 분류 기준, 통치 수단, 정당화 언어, 실제 변화, 저항·균열을 바탕으로 비교할 만한 역사적 쟁점을 찾는다. 특정 역사 장면을 다른 장면과 같다고 단정하지 않고, 사건·제도·생활상·사조·예술·종교·관습의 닮은 점과 다른 점을 함께 확인하는 학습 흐름으로 사용한다.

## 10A 품질 기준

현재 요청문은 사회·정치·제도·역사 분석을 우선한다. ChatGPT가 먼저 현실의 제도, 시장, 행정, 법적 권한, 행위자의 이해관계, 외부 조건이 바꾸는 비용과 책임 구조를 분석하고, 사용자가 보는 JSON에는 세계사와 비교할 쟁점만 짧고 밀도 있게 쓰게 만든다.

좋은 응답은 유명한 역사 사건 이름을 바로 붙이지 않는다. 현실의 출발점, 실제 권한 경로, 외부 조건의 역할, 비용 이전, 오류 비용 부담자, 저항과 균열을 설명한 뒤 비교 가능한 역사 렌즈로 넘어간다.

10C 기준에서는 1차 `axis-finder`의 `overview`를 최대 4문장, 각 `bridge`와 `whyThisAxisMatters`를 최대 5문장으로 압축한다. 2차 요청은 앱으로 되돌릴 JSON이 아니라 ChatGPT 창에서 바로 읽는 학습 해설이다. 따라서 사건, 제도, 생활상, 사조, 예술, 종교, 관습을 가로질러 전후 맥락, 닮은 점, 결정적인 차이, 확인할 키워드를 사람이 읽기 좋은 한국어 문단으로 받는다.

ChatGPT가 생성 과정 메타 문장을 쓰면 앱에 붙여 넣기 전에 다시 요청한다. 예를 들어 “제공된 편집 브리프에 근거한다”, “사용자 입력에 충실하게 작성했다”, “요청된 JSON 구조를 따랐다” 같은 문장은 결과 내용이 아니다.

요소 사용률보다 인과관계의 선명함을 우선한다. 유명하거나 인상적인 역사 비유를 먼저 고른 뒤 그 비유에 맞는 정책 경로를 사후 발명하면 안 된다. 사회 압력은 가능하면 사용하되, 약한 중간 제도를 새로 만들어야만 연결된다면 보류하는 편이 낫다.

모든 쟁점은 입력된 영향을 받는 집단을 중심에 둔다. 사회 압력을 쓰지 않은 쟁점도 집단과 다른 핵심 요소들이 강하게 연결된다면 정식 쟁점이 될 수 있다. 반대로 독립적으로 유명한 역사 비유라도 현재 입력에서 가장 적은 비약으로 도출되지 않으면 좋은 쟁점이 아니다.

이 방식은 별도 OpenAI API 키나 결제 설정 없이, 현재 ChatGPT 대화창을 사용해 결과 품질을 시험하기 위한 흐름이다.

## 사용 순서

1. `0단계 · 탐색 범위`에서 필요하면 우선 살펴볼 시대·지역 범위를 힌트로 적는다. 비워두어도 된다.
2. 앱에서 빠른 시작 템플릿을 적용하거나, 조건, 구조, 작동과 결과 요소를 선택하거나 직접 입력한다. 모든 칸을 채울 필요는 없고, 하나의 조건만 적어도 1차 요청을 만들 수 있다.
3. `ChatGPT에 세계사 렌즈 요청 만들기`를 누른다.
4. `일반 ChatGPT용 전체 요청 복사`를 누른다.
5. ChatGPT 대화창을 열고 그대로 붙여 넣는다.
6. ChatGPT가 반환한 `axis-finder` JSON 전체를 복사한다.
7. 앱의 `ChatGPT 응답 붙여넣기` 영역에 붙여 넣는다.
8. `역사적 쟁점 적용`을 누르면 2~3개의 역사적 쟁점 카드가 표시된다.
9. 마음에 드는 쟁점의 `이 쟁점을 ChatGPT에서 더 읽기`를 눌러 2차 학습 프롬프트를 복사한다.
10. ChatGPT에 붙여 넣고, 반환된 해설은 ChatGPT 창에서 그대로 읽는다.
11. 입력을 수정했다면 다시 요청문을 만든다.

첫 응답은 최종 설명이 아니다. 모든 요소를 하나의 이야기로 합치기 전에, 어떤 역사적 쟁점으로 읽을 수 있는지 분리해서 보는 단계다.

1차 `axis-finder` JSON의 각 쟁점에는 역사 공부용 필드가 포함된다. `historicalCases`는 사건뿐 아니라 제도, 생활상, 사조, 예술, 종교, 관습, 물질문화까지 포함할 수 있고, 각 사례는 전후 맥락과 읽을 점, 검증 키워드를 함께 가져야 한다. `similarities`와 `differences`는 단순 동일시를 막기 위한 비교 기준이며, `humanBehaviorPattern`과 `futureInsight`는 과거를 앞으로의 사회를 읽는 관찰로 연결하기 위한 필드다.

## 주의

- 앱 안의 로컬 브리프는 최종 해석이 아니다.
- 로컬 규칙은 창의성을 평가하지 않는다.
- 낯선 연결은 오류가 아니라 ChatGPT가 해석해야 할 부분이다.
- 1차 ChatGPT 응답은 JSON만 붙여 넣는 것이 가장 좋다.
- 1차 JSON 뒤에 참고문헌 목록이나 각주가 붙은 경우, 앱은 첫 번째 JSON 객체만 추출해 적용한다.
- ```json 코드펜스가 포함된 응답도 붙여 넣을 수 있다.
- ChatGPT가 최신 `axis-finder` 구조를 지키지 않으면 앱은 누락 필드 예시와 `다시 요청 문구 복사` 버튼을 보여준다. 이 문구를 ChatGPT에 붙여 넣어 JSON만 다시 출력하게 한다.
- ChatGPT가 확인하지 않은 참고문헌 번호, URL, 책 제목, 논문 제목, 기사 제목, 저자명, 쪽수, 직접 인용문을 만들면 그대로 믿지 말고 검증 키워드로 다시 확인한다.
- 현재 프롬프트는 `[1]`, `[2]` 같은 가짜 각주나 참고문헌 목록을 붙이지 말고, 확실한 사례와 비교 가설을 구분하도록 요구한다.
- 쟁점 카드의 `확인 키워드 복사`는 사례별 검증 키워드를 모으는 기능이다. 이 목록은 출처 목록이 아니며, 사용자가 사전·백과·도서관·논문 검색에서 사실관계를 재확인하기 위한 검색 출발점이다.
- 2차 답변은 앱에 붙여 넣지 않는다.
- 앱에 붙여 넣는 응답의 `version`은 `"1.0"`이어야 한다.
- 앱에 붙여 넣는 응답의 `interpretationMode`는 `axis-finder`만 사용한다.

## 선택적 API 개발 모드

`?apiAi=1`을 URL에 붙이면 기존 로컬 Node 서버의 `/api/interpret` 흐름을 사용할 수 있다. 기본 권장 방식은 수동 ChatGPT 연결이다.

## 2차 학습 프롬프트

1차 `axis-finder` JSON을 적용하면 쟁점 카드가 나타난다. 사용자가 한 쟁점의 `이 쟁점을 ChatGPT에서 더 읽기`를 누르면 같은 결과 화면 안에 선택한 쟁점 요약과 2차 프롬프트 복사 버튼이 열린다.

2차 프롬프트는 JSON을 요구하지 않는다. ChatGPT는 핵심 요지, 비교할 세계사 장면, 앞뒤 맥락, 닮은 점과 결정적인 차이, 확인할 키워드, 더 공부할 질문을 사람이 읽는 해설로 답한다. 확인하지 않은 책·논문·URL·각주는 만들지 않고, 확실히 아는 대표 키워드와 사용자가 검증할 검색어만 남기도록 요구한다.

2차 패널에서 할 수 있는 일:

- `2차 학습 프롬프트 다시 복사`: 현재 선택된 쟁점의 2차 요청문을 다시 복사한다.
- `쟁점 선택 취소`: 선택 상태를 초기화하고 1차 쟁점 카드 화면으로 돌아간다.
