# 10A 프롬프트 품질 발견 기록

## 목적

이번 단계는 기능 추가가 아니라 ChatGPT 요청문의 사고 순서를 교정하는 작업이다. 목표는 사용자의 조건을 세계사 학습에 도움이 되는 비교 질문으로 바꾸고, 인간 사회의 반복되는 행동 양식과 제도 논리를 근거 중심으로 이해하는 것이다.

## 발견한 반복 문제

4개 입력 조합에서 여러 방향을 가정해 점검하면 다음 문제가 반복될 가능성이 높다.

- 사회·정치·제도 분석보다 이야기식 결론이 먼저 나온다.
- 조건부 권리, 자동 배정, 점수 공식이 기본 답안처럼 반복된다.
- 외부 조건이 제도 변화를 실제로 일으키는 방식보다 배경 소품처럼 쓰인다.
- 기업, 보험사, 플랫폼, 국제기구가 현실적 권한 경로 없이 국가 권한을 행사한다.
- 사회문제와 영향을 받는 집단이 막연한 위기 서사로 희석된다.
- 현실 제도에서 역사 비교 쟁점으로 이동하는 2~4단계 과정이 생략된다.
- 새 기관이나 새 제도 이름을 발명하는 것으로 연결의 빈틈을 메운다.
- 비용과 오류를 누가 부담하는지 흐려진다.
- 유명한 역사 사례를 먼저 정하고 입력을 거기에 맞춘다.

## 10A 교정 내용

1차 `axis-finder` 프롬프트는 각 쟁점을 쓰기 전에 다음 순서를 따르도록 강화했다.

1. 현실의 출발점
2. 사회문제와 영향을 받는 집단
3. 행위자의 이해관계
4. 실제 권한 경로
5. 외부 조건의 실제 역할
6. 제도 변환 경로
7. 비용 이전
8. 비대칭
9. 중심 모순
10. 역사 렌즈 번역

2차 학습 프롬프트는 선택된 쟁점 하나만 바텀업하며, JSON을 요구하지 않고 ChatGPT 창에서 바로 읽을 수 있는 세계사 해설을 쓰도록 강화했다. 사례별 전후 맥락, 닮은 점, 결정적인 차이, 확인할 키워드를 반드시 포함한다.

## 반복 공식 통제

다음 장치는 입력에 있거나 결과상 필수일 때만 사용한다.

- 위험지수
- 면제권
- 지역 축소가 다시 지역 축소를 낳는 악순환
- 통합 계정에서 이탈하면 권리 접근이 제한되는 구조
- 전문 서비스가 고가화되는 구조
- 체납 기록이 권리 접근에 영향을 주는 공식
- 알고리즘 자동 배정

쟁점마다 서로 다른 권력 작동 방식을 우선한다. 예를 들어 가격과 보험, 계약과 구독, 허가와 면허, 기반시설 접근권, 소유권과 상속권, 노동과 돌봄 의무, 공간 배치와 이주, 인증과 시험, 공적 의무의 민간 위상, 국제 책임의 외주화, 대기기간과 순번, 평판과 공개 기록, 법적 책임 분산, 서비스 호환성과 데이터 이동성 등을 구분한다.

## 외부 조건 반사실 검증

외부 조건 요소는 다음 질문을 통과해야 `usedElements`에 들어갈 수 있다.

- 이 조건이 무엇을 더 싸게, 빠르게, 대규모로, 자동으로, 측정 가능하게 만드는가?
- 이 조건이 권한과 책임을 어떻게 분리하는가?
- 이 조건을 제거해도 쟁점의 인과관계가 거의 같은가?

외부 조건을 제거해도 쟁점이 거의 같다면 해당 요소는 `usedElements`가 아니라 `deferredElements`로 보류한다.

## 행위자 권한 점프 방지

기업, 보험사, 플랫폼, 국제기구가 입력에 없는데 갑자기 국가 권한을 직접 갖는 것으로 쓰지 않는다. 먼저 다음 경로를 검토한다.

- 국가의 법적 허가
- 필수 인프라 독점
- 계약과 보험 조건
- 인증과 표준
- 데이터 독점
- 서비스 접근권
- 법적 결정과 실무 집행의 분리
- 기존 기관 간 책임 이전

## 사회문제와 집단 희석 방지

모든 정식 쟁점은 입력된 영향을 받는 집단을 중심에 둔다. 사회문제는 실제 중심 인과관계에 참여할 때만 사용한다. 2~3개 쟁점 중 최소 1개는 사회문제를 직접 사용하는 현실적 경로를 우선 탐색하되, 경로가 약하면 억지로 만들지 않는다.

## usedElements 강화

요소를 `usedElements`에 넣으려면 다음 중 하나를 실제로 바꿔야 한다.

- 문제의 원인
- 행위자의 이해관계
- 권력 행사 방식
- 측정 기준
- 권리 변환
- 피해 발생 경로
- 통찰을 정당화하는 논리
- 계층적 비대칭

단순 언급, 배경 소품, 분위기 장식은 사용으로 인정하지 않는다.

## generatedSuggestions 축소

`generatedSuggestions`는 최대 3개로 제한한다. 서로 다른 type만 사용하고, 행정 문구는 `possibleAdministrativeLanguage` type으로 최대 1개만 쓴다. 가능한 경우 검증할 역사 키워드, 비교할 제도, 또는 차이와 한계를 남긴다. 선동적 문구나 새 JSON 필드는 추가하지 않는다.

## 계속 볼 평가 기준

- 결과가 이야기보다 분석에 가까운가?
- 분석 내용이 전체 응답의 중심을 차지하는가?
- 역사 비교가 입력에서 직접 파생되는가?
- 각 쟁점의 권력 작동 방식이 다른가?
- 조건부 권리 공식이 자동 반복되지 않는가?
- 사회문제와 영향을 받는 집단이 유지되는가?
- 외부 조건을 제거했을 때 차이가 설명되는가?
- 외부 조건이 장식이면 보류되는가?
- 기업과 국제기구의 권한 경로가 현실적으로 설명되는가?
- 기존 제도에서 변형되는 단계가 보이는가?
- 비용과 실패의 부담자가 드러나는가?
- 새 기관 이름으로 연결의 빈틈을 메우지 않는가?
- 역사 사례의 닮은 점과 다른 점이 함께 보이는가?

## 10D 마지막 교훈

강한 중심 모순 자체와 입력 조합의 적합성은 별개다. 특정 비교가 독립적으로 흥미롭더라도 현재 입력과 연결하기 위해 대규모 정책 전환을 새로 발명해야 한다면 좋은 세계사 렌즈가 아니다.

앞으로는 요소 사용률보다 인과관계의 명확성을 우선한다. 사회문제는 여러 쟁점에 강제하지 않고, 최소 한 쟁점에서 정확하게 쓰는 편이 낫다. 현실적인 연결 경로가 없다면 사회문제를 `deferredElements` 또는 한계 설명으로 남긴다.

좋은 세계사 렌즈는 결론이 유명해서가 아니라, 입력 조건에서 비교 질문으로 넘어가는 경로가 단순하고 검증 가능해야 한다.
